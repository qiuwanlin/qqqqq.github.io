css-BFC

Posted on 2018-09-10 | In css

BFC的定义

  • Bloack Formatting Context 块格式化上下文。

CSS规范中对 BFC 的描述：

浮动、绝对定位元素、非块盒的块容器（例如，inline-blocks、table-cells和table-captions）和’overflow’不为’visible’的块盒会为它们的内容建立一个新的块格式化上下文。
在一个BFC中，盒在竖直方向一个接一个地放置，从包含块的顶部开始。两个兄弟盒之间的竖直距离由’margin’属性决定。同一个BFC中的相邻块级盒之间的竖直margin会合并。
在一个BFC中，每个盒的left外边（left outer edge）挨着包含块的left边（对于从右向左的格式化，right边挨着）。即使存在浮动（尽管一个盒的行盒可能会因为浮动收缩），这也成立。除非该盒建立了一个新的块格式化上下文（这种情况下，该盒自身可能会因为浮动变窄）。

MDN 对 BFC 的描述：

一个BFC是Web页面的可视化CSS渲染出的一部分。它是块级盒布局出现的区域，也是浮动层元素进行交互的区域。
display:flow-root是专门用来触发BFC没别的用的新增属性。
一个BFC包括创建它的元素内部所有内容，除了被包含于创建新的块级格式化上下文的后代元素内的元素。(父元素是BFC就会包住子元素，如果子元素也是BFC那父元素就管不着孙子元素)
BFC对于定位 (参见 float) 与清除浮动 (参见 clear) 很重要。定位和清除浮动的样式规则只适用于处于同一BFC内的元素。浮动不会影响其它BFC中元素的布局，并且清除浮动只能清除同一BFC中在它前面的元素的浮动。

它没有定义,只有特征/功能。

我不知道什么是BFC，但你写出样式我就知道这是不是BFC。

BFC的功能

  1. 用BFC包住浮动元素
    子元素浮动后父元素没有高度。
    看CSS规范得知，若父元素也浮动或绝对定位或变为非块盒的块容器或overflow:auto就可以包住子元素了，父元素即形成一BFC。
  2. 元素间划清界限
    比如AB为级别相同元素，A元素左浮后会跑到B上面，像是B包住了A，这时B也触发BFC（比如overflow:auto）则会划清界限。

css flex布局

Posted on 2018-09-08 | In css

flex container的属性

flex-direction（方向）

默认 row：横向
row-reverse ：反向
column ：竖向
column-reverse :竖反向

flex-wrap（换行）

默认 nowrap :不换行
wrap :自动换行
wrap-reverse:自动换行且反向。

  • flex-flow是上面那俩的简写

justify-content（主轴方向对齐方式）

默认 flex-start: 元素和容器的左端对齐。
flex-end: 元素和容器的右端对齐。
center: 元素在容器里居中。
space-between:元素之间保持相等的距离。
space-around:元素周围保持相等的距离。

align-items（侧轴对齐方式）

默认 stretch: 元素被拉伸以填满整个容器。
flex-start: 元素与容器的顶部对齐。
flex-end: 元素与容器的底部对齐。
center: 元素纵向居中。
baseline: 元素在容器的基线位置显示。（用得少）

align-content （多行/列内容对齐方式）（用得少）

flex-start: 多行都集中在顶部。
flex-end: 多行都集中在底部。
center: 多行居中。
space-between: 行与行之间保持相等距离。
space-around: 每行的周围保持相等距离。
stretch: 每一行都被拉伸以填满容器。

  • 这可能容易与上一个属性混淆，但align-content决定行之间的间隔，而align-items决定元素整体在容器的什么位置。只有一行的时候align-content没有任何效果。

flex item的属性

flex-grow（增长比例（空间过多时））

假设有三个子元素：

  • flex-shrink （收缩比例（空间不够时））
    这个属性一般不用，用法和上一个类似。

flex-basis（默认大小）

这个属性一般不用。
不写就是原始大小，写了就按你写的来显示。

  • flex属性是以上三个属性的简写。
    写法如 flex:1 2 100px

order（顺序）

改变元素展示顺序。
元素的属性默认值为0，但是我们设置这个属性为正数或负数。
代替双飞翼布局的原始套路。

align-self（自身对齐方式）

对抗父元素的align-items属性。
属性值与其相同。

css动画

Posted on 2018-08-13 | In css

transition

在CSS3引入transition之前css没有时间轴，所有的状态变化都是瞬间完成，transition是指定状态变化所需要的时间。

  1. 目前，各大浏览器（包括IE 10）都已经支持无前缀的transition，所以transition已经可以很安全地不加浏览器前缀。
  2. transition需要明确知道，开始状态和结束状态的具体数值，才能计算出中间状态，none到block之类的是不行的。
  3. transition是一次性的，不能重复发生，除非一再触发。

指定属性&delay

我们可以指定transition适用的属性，比如只适用于height，这样一来，只有height的变化需要1秒实现，其他变化（主要是width）依然瞬间实现。
在同一行transition语句中，可以分别指定多个属性。

但这样height和width的变化是同时进行的，跟不指定它们没有差别，如果希望让height先变化，结束以后width再发生变化，这样就要为width指定一个delay参数。width在1秒之后再开始变化，也就是延迟（delay）1秒

transition-timing-function

  • transition的状态变化速度（又称timing function），默认不是匀速的，而是逐渐放慢，这叫做ease。

除了ease以外，其他模式还包括：
linear：匀速
ease-in：加速
ease-out：减速
cubic-bezier函数：自定义速度模式 预览

语法

transition的完整写法如下

这其实是一个简写形式，可以单独定义各个属性
transition-property: height;
transition-duration: 1s;
transition-delay: 1s;

transition-timing-function: ease;

animation

transition 比较简单，animation可以帮我们实现复杂的动画。

animation-fill-mode

  • 动画结束以后，会立即从结束状态跳回到起始状态。
  • 如果想让动画保持在结束状态，需要使用animation-fill-mode属性。

animation-fill-mode可以定义下列值：
none：默认值，回到动画没开始时的状态
forwards：让动画停留在结束状态
backwards：让动画回到第一帧的状态
both: 根据animation-direction轮流应用forwards和backwards规则。

animation-direction

  • 动画循环播放时，每次都是从结束状态跳回到起始状态再播放。animation-direction属性可以改变这种行为。
  • animation-direction指定了动画播放的方向，最常用的值是normal和reverse。浏览器对其他值的支持情况不佳，慎用。
    默认情况animation-direction等于normal。

还可以取alternate（ 动画应该轮流反向播放。）、reverse（动画反向播放。）、alternate-reverse等值。

animation-play-state

有时动画播放过程中会突然停止。这时默认行为是跳回到动画的开始状态，如果想让动画保持突然终止时的状态，就要使用animation-play-state属性。

语法

同transition一样，animation也是一个简写形式。

keyframes关键字用来定义动画的各个状态，它的写法相当自由，0%可以用from代表，100%可以用to代表，如果省略某个状态，浏览器会自动推算中间状态。
甚至，可以把多个状态写在一行。

另外，浏览器从一个状态向另一个状态过渡是平滑过渡。steps函数可以实现分步过渡。
